# Stockbook — 股票交易記帳 Web App

這是一個專為股票交易設計的記帳與庫存管理系統，特別針對台股（如元大台灣50，0050）設計了靈活的交易配對與回補管理邏輯。

---

## 🛠️ 維護者資訊

本專案由 AI 程式開發助手 **Antigravity**（由 Google DeepMind 設計與研發）與使用者協作共同開發與維護。

## 📋 系統維護與更新紀錄 (2026-07-07)

在此次維護中，我們針對系統功能進行了以下優化與 bug 修復：

### 1. 新增放空回補（先賣後買）配對機制
- **原問題**：系統原設計僅支援常規的「先買後賣（Long）」配對，當使用者在下行趨勢中先賣出股票避險、再於更低價格補回時，系統無法正確將該賣出單與隨後的買入（回補）單做配對，導致無法實現放空利差，且會錯誤地將賣出單與早前的長期庫存部位做扣抵。
- **解決方案**：
  - 新增「Phase 1：放空回補配對」邏輯。在計算已實現損益時，會優先檢索標記為「回補（REBUY）」用途且關聯了特定賣出單 ID 的買入單。
  - 將這部分買入單的股數優先與對應賣出單做沖銷，計算出放空價差損益並產生 realized profit/loss，且扣除這些已消耗的股數。
  - 隨後進行「Phase 2：常規先買後賣配對」，這能確保原有的長期庫存部位完全不受短線避險（放空回補）的影響。

### 2. 實作交易紀錄編輯功能（支援二次確認）
- **原問題**：原系統只提供交易的「刪除」功能，缺乏直接編輯交易紀錄的欄位，對輸入錯誤資料時極度不便。
- **解決方案**：
  - 在交易明細列表（電腦版/手機版）以及庫存操作明細中，全面新增「編輯」按鈕。
  - 點擊「編輯」會直接將該交易的所有欄位資訊（日期、帳戶、代號、價格、股數、手續費、稅金、配對關係與備註等）回填至記帳表單。
  - 為防使用者不慎改動舊交易導致歷史庫存算亂，在點擊「確認修改」存檔時，系統會強制彈出**兩次確認提示框**：
    1. *第一關*：「您確定要修改此筆交易紀錄嗎？」
    2. *第二關*：「修改交易將會重新計算所有庫存與損益，請再次確認！」
  - 兩次均確認後才會寫入更新並重算。

### 3. 修復手機版記帳彈窗「確認按鈕」遭遮擋問題
- **原問題**：當使用者在手機畫面中新增買入或展開配對 picker 時，因欄位長度超出或手機虛擬鍵盤彈出，表單底部的「確認/儲存」按鈕會被遮擋在可視範圍之外，無法進行點擊。
- **解決方案**：
  - 重構了 CSS 中的彈窗版面樣式。將記帳彈窗限制在 `max-height: calc(100svh - 40px)` 內。
  - 將彈窗設定為 `flex` 佈局，使表單頭部與底部的「取消/確認修改」按鈕列固定（`flex-shrink: 0`）。
  - 中間的欄位輸入區設為獨立滾動（`overflow-y: auto`）。這使得不論表單多長，按鈕群都會牢牢固定在手機螢幕的最下方，完美解決遮擋問題。

### 4. 解決 Firebase 同步失敗問題（突破 Firestore 1MB 限制）
- **原問題**：當使用者的交易紀錄、對帳紀錄或審計日誌（Audit Logs）累積較多時，同步會發生失敗。這是因為 Firebase Firestore 的單一文件（Document）最大尺寸限制為 **1,048,576 bytes (1MB)**，而原系統是將整個帳本 JSON 狀態一次寫入單一文件中。
- **解決方案**：
  - 導入瀏覽器原生的 `CompressionStream` 與 `DecompressionStream` 對 JSON 資料進行 **Gzip 壓縮**，可將檔案體積大幅縮減約 90%。
  - 將壓縮後的資料轉換為 Base64 字串，並以 **800KB** 為上限進行**切片（Chunking）**。
  - 主文件 `stockLedgers/{namespace}` 僅儲存分塊數量等元資料，實際資料分塊寫入子集合中：`stockLedgers/{namespace}/chunks/chunk_{i}`。
  - 載入時，若發現舊版 uncompressed 資料會直接讀取；若是新版分塊資料則會拉取所有分塊拼接、解壓還原，確保**向後相容**。

## 🚀 本機執行指南

要在您的本機環境下預覽與測試此 Web App，請按照以下步驟執行：

1. 開啟終端機（Terminal/PowerShell），並切換至專案根目錄。
2. 執行以下指令啟動本機靜態伺服器：
   ```bash
   npx http-server public
   ```
3. 啟動後，在瀏覽器開啟視窗中提示的本機網址（例如：`http://127.0.0.1:8080`），即可直接使用與測試所有功能！
